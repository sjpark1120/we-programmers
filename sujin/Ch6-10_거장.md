# 2부 거장

## 6장 에츠허르 다익스트라: 첫 번째 컴퓨터 과학자

- 다익스트라
  - 구조적 프로그래밍의 아버지
  - GOTO문 사용을 지양해야 한다고 주장
  - 세마포어 개념 고안, 다양한 알고리즘 개발
  - ALGOL 60 컴파일러 초기 버전 개발 참여
  - 초기 멀티프로그래밍 운영체제 설계

  - ARRA
    - 프로그램 내장 방식의 전기기계식 컴퓨터
    - 소프트웨어를 하드웨어와 독립적으로 개발하려는 시도 -> 의존성 역전

  - ARMAC (1956)
    - ARRA 계열의 발전된 컴퓨터
    - 다익스트라 알고리즘 (최단 경로)
    - 최소 신장 트리 관련 개념 발전

- algol과 x1
  - 프로그래밍 언어 구조는 하드웨어의 물리적 구조와 분리되어야 함
    (당시 FORTRAN, COBOL은 하드웨어 의존적이었다는 점에서 반대되는 주장)
  - 실행 속도는 다소 느릴 수 있음
  - 대신 프로그래밍 시간 단축 + 실수 방지
  - ALGOL
    - 범용 프로그래밍 언어
    - 구조적 프로그래밍 개념의 기반 제공
  - X1
    - 하드웨어 인터럽트 지원
    - 인덱스 레지스터, 포인터 사용 가능
    - 이중 프로그래밍 지원
  - 추상화 경계
    - 컴파일러: ALGOL → 중간 코드(P-code) 변환
    - 런타임 시스템: P-code 해석 및 실행
    - → 오늘날 가상 머신(VM) 구조와 유사

    - 컴파일러는 더 이상 하드웨어를 직접 신경 쓰지 않아도 됨
    - 프로그램의 올바름 보장이 성능 최적화보다 더 중요하다는 관점 제시

- 소프트웨어 위기와 컴퓨터 과학의 등장
  - 프로그램 규모 증가로 복잡도 급증 → 소프트웨어 위기 발생
  - 체계적인 접근 필요 → 컴퓨터 과학이라는 학문 등장

  - 멀티프로그래밍
  - 시스템을 블랙박스 계층으로 분리

  - 계층 구조 (Layered System)
    1. 레이어 0: 프로세서 할당
    2. 레이어 1: 메모리 제어
    3. 레이어 2: 시스템 콘솔 제어
    4. 레이어 3: 입출력 및 버퍼링 제어
    5. 레이어 4: 사용자 프로그램 실행

  - 세마포어
    - 하나의 정수값으로 표현되는 동기화 도구
    - P 연산 (wait)
      - 값이 0 이하 → 프로세스 차단
      - 값이 양수 → 1 감소 후 실행
    - V 연산 (signal)
      - 값 증가
      - 대기 중인 프로세스 하나를 깨움

  - 임계 구역 (Critical Section)
    - 공유 자원을 다루는 코드 영역
    - 동시에 실행되면 안 됨 (동시 업데이트 방지)

  - 불가분 연산 (Atomic Operation)
    - 중간에 인터럽트 없이 반드시 한 번에 실행되어야 하는 연산

- 구조적 프로그래밍
  - 프로그램을 순차적인 프로시저의 집합으로 바라봄
  - GOTO문을 제거하고 구조적으로 작성

  - 기본 구조
    - 순차 (Sequence)
    - 선택 (Selection)
    - 반복 (Iteration)

- 프로그래밍은 수학인가?
  - 수학: 무엇이 “옳은지”를 증명하는 학문
  - 소프트웨어: 무엇이 “잘못되었는지”를 발견하는 과정

  → 소프트웨어는 수학이라기보다 과학에 가까운 영역

## 7장 니가드와 달: 첫 번째 OOPL

- 크리스텐 니가드
- 올레 요한 달
- SIMULA와 객체지향(OO)
  - SIMULA 67: 최초의 객체지향 언어
  - 몬테카를로 시뮬레이션을 위한 언어에서 출발

  - 개념 구성
    1. 커스터머
       - 여러 속성 데이터를 가지는 수동적 객체 (데이터 중심)
    2. 스테이션
       - 내부에 큐를 가지고 있음
       - 커스터머를 처리 후 다른 스테이션으로 전달
         이산 이벤트 네트워크: 커스터머가 스테이션의 큐에 삽입되는 모든과정은 이벤트로 트리거되는 방식
  - 이산 이벤트 네트워크
    - 커스터머가 큐에 들어가고 이동하는 모든 과정이 이벤트 기반으로 동작

  - 초기 구조
    - ALGOL 60 기반 전처리 형태
    - SIMULA 코드를 ALGOL 60 코드로 변환

- SIMULA 1
  - 기존 ALGOL 60
    - 블록 구조 언어
    - 함수는 지역변수를 가지며, 스택에 저장됨

  - 시뮬레이션 모델에서 스테이션과 커스터머 역할을 하는 ALGOL60 블록들을 생성하는 전처리기
  - 문제 큐, 선입선출구조 > ALGOL60의 블록을 스택이 아니라 힙에 할당
    - ALGOL의 블록 개념은 유지
    - 하지만 블록을 스택이 아니라 힙에 할당
    - 가비지 컬렉션이 가능한 구조로 변경

    - 다양한 객체가 서로 다른 시점에 생성/소멸 -> 스택 대신 힙 기반 객체 구조 도입
      -> 지역변수 + 객체 상태를 함께 유지 가능

    - 이 과정이 객체지향 프로그래밍의 시작점

  - 가비지 컬렉터
    - 참조 카운팅 기반
    - 스택과 힙 사이의 절충적 구조
    - 메모리 단편화 문제 완화

    - 커스터머와 스테이션간 상호작용 : 준병렬 처리 개념

    - 힙 기반 메모리 관리
      → 객체 생명주기를 유연하게 관리

    - 결과
      → 능동적인 객체 개념 + 객체 간 상호작용 정립
      → 객체지향 프로그래밍의 핵심 개념 형성

  - 스테이션과 커스터머는 결국 시뮬레이션안에서 동시에 작동할수있는 여러 종류의 구성요소 중 하나 일뿐
    커스터머, 스테이션 >> 액티비티

- 객체지향 개념 정립
  - 클래스 / 서브클래스
  - 객체 (인스턴스)
  - 타입과 클래스 개념

  -> 객체 + 상태 + 행동 + 상호작용  
  -> 현대 객체지향 프로그래밍의 핵심 개념 확립

## 8장 존 케메니: 모두를 위한 첫 번째 언어, BASIC

- 존 케메니
  - 프로그래밍 언어는 사람 중심으로 설계되어야 한다고 주장

- 당시 상황
  - MIT의 컴퓨터를 사용하기 위해 2주 대기
  - 한 번에 한 사람만 사용 가능 (동시 사용 불가)

  → 비효율 해결을 위해 타임 쉐어링(Time-Sharing) 개념 고안
  - 다트머스 타임셰어링 시스템
    - 여러 사용자가 동시에 하나의 컴퓨터를 사용하는 방식
    - BASIC 컴파일러 탄생

- BASIC
  - 초보자용 범용 기호 명령어 코드 (Beginner’s All-purpose Symbolic Instruction Code)
    - 초보자용 범용 언어
    - 쉽고 직관적
    - 대화형(interactive) 방식
    - 빠르고 추상적인 언어
      - 전신타자기로 쉽게 입력 가능
  - 문법이 단순 -> 컴파일러 구현도 쉬움
  - 대화형 프로그래밍 환경을 대중화

- 컴퓨터 키즈
  - BASIC 인터프리터의 확산
    ->개인용 컴퓨터 시대와 맞물려 빠르게 퍼짐

  - 컴퓨터를 전문가가 아닌 일반 사람들이 사용하는 시대의 시작

- 모든 사람이 컴퓨터를 사용할 수 있고, 사용해야 한다고 믿음

케메니의 예견

- 기술 및 사회 변화
  - 전화, 단말기, 네트워크 발전
  - 교육 변화
  - 사생활 문제 (빅브라더)
  - 무어의 법칙 종말

## 9장 주디스 앨런

- 쥬디스 슐츠
  - 1957년 가을, 오리건 주립대학교 수학과에서 최초로 ALWAC III-E 컴퓨터 도입
  - 해당 컴퓨터 프로그래밍 강의를 듣는 유일한 여학생

- ECP-18
  - 담당 교수가 직접 설계한 컴퓨터
  - 쥬디스는 이 컴퓨터를 접하게 됨
    - 전업주부 상태였지만 ECP-18의 어셈블러 제작 요청을 받음
    - 2패스 방식(2-pass assembler) 사용

  - 여성이라는 이유로 차별과 폭력 경험
