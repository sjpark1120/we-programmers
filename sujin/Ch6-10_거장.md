# 2부 거장

## 6장 에츠허르 다익스트라: 첫 번째 컴퓨터 과학자

- 다익스트라
  - 구조적 프로그래밍의 아버지
  - GOTO문 사용을 지양해야 한다고 주장
  - 세마포어 개념 고안, 다양한 알고리즘 개발
  - ALGOL 60 컴파일러 초기 버전 개발 참여
  - 초기 멀티프로그래밍 운영체제 설계

  - ARRA
    - 프로그램 내장 방식의 전기기계식 컴퓨터
    - 소프트웨어를 하드웨어와 독립적으로 개발하려는 시도 -> 의존성 역전

  - ARMAC (1956)
    - ARRA 계열의 발전된 컴퓨터
    - 다익스트라 알고리즘 (최단 경로)
    - 최소 신장 트리 관련 개념 발전

- algol과 x1
  - 프로그래밍 언어 구조는 하드웨어의 물리적 구조와 분리되어야 함
    (당시 FORTRAN, COBOL은 하드웨어 의존적이었다는 점에서 반대되는 주장)
  - 실행 속도는 다소 느릴 수 있음
  - 대신 프로그래밍 시간 단축 + 실수 방지
  - ALGOL
    - 범용 프로그래밍 언어
    - 구조적 프로그래밍 개념의 기반 제공
  - X1
    - 하드웨어 인터럽트 지원
    - 인덱스 레지스터, 포인터 사용 가능
    - 이중 프로그래밍 지원
  - 추상화 경계
    - 컴파일러: ALGOL → 중간 코드(P-code) 변환
    - 런타임 시스템: P-code 해석 및 실행
    - → 오늘날 가상 머신(VM) 구조와 유사

    - 컴파일러는 더 이상 하드웨어를 직접 신경 쓰지 않아도 됨
    - 프로그램의 올바름 보장이 성능 최적화보다 더 중요하다는 관점 제시

- 소프트웨어 위기와 컴퓨터 과학의 등장
  - 프로그램 규모 증가로 복잡도 급증 → 소프트웨어 위기 발생
  - 체계적인 접근 필요 → 컴퓨터 과학이라는 학문 등장

  - 멀티프로그래밍
  - 시스템을 블랙박스 계층으로 분리

  - 계층 구조 (Layered System)
    1. 레이어 0: 프로세서 할당
    2. 레이어 1: 메모리 제어
    3. 레이어 2: 시스템 콘솔 제어
    4. 레이어 3: 입출력 및 버퍼링 제어
    5. 레이어 4: 사용자 프로그램 실행

  - 세마포어
    - 하나의 정수값으로 표현되는 동기화 도구
    - P 연산 (wait)
      - 값이 0 이하 → 프로세스 차단
      - 값이 양수 → 1 감소 후 실행
    - V 연산 (signal)
      - 값 증가
      - 대기 중인 프로세스 하나를 깨움

  - 임계 구역 (Critical Section)
    - 공유 자원을 다루는 코드 영역
    - 동시에 실행되면 안 됨 (동시 업데이트 방지)

  - 불가분 연산 (Atomic Operation)
    - 중간에 인터럽트 없이 반드시 한 번에 실행되어야 하는 연산

- 구조적 프로그래밍
  - 프로그램을 순차적인 프로시저의 집합으로 바라봄
  - GOTO문을 제거하고 구조적으로 작성

  - 기본 구조
    - 순차 (Sequence)
    - 선택 (Selection)
    - 반복 (Iteration)

- 프로그래밍은 수학인가?
  - 수학: 무엇이 “옳은지”를 증명하는 학문
  - 소프트웨어: 무엇이 “잘못되었는지”를 발견하는 과정

  → 소프트웨어는 수학이라기보다 과학에 가까운 영역
