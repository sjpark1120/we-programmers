# 4부 미래

## 16장 프로그래밍 언어

- 오늘날 C, C++, 자바, C#, 자바스크립트, 루비, 파이썬, 오브젝트C, 스위프트, 코틀린, 다크, 러스트, 엘름, GO, PHP 등 매우 다양한 언어가 존재
- 시간이 지날수록 새로운 언어에서 근본적인 혁신은 줄어드는 흐름  
  -> 다른 공학 분야처럼 표준 기호와 체계가 정착되는 방향으로 발전 해야 한다
  -> 결국 하나의 언어로 통일될 것 (리스프 계열 언어)

- 타입 시스템
  - 정적 타입 / 동적 타입 각각 장단점 존재
  - 타입 검사를 엄격하게 하되, 그 시점을 런타임으로 미루는 방식에 대한 고민

- 리스프(Lisp)
  - 문법이 극도로 단순하지만 표현력은 매우 풍부
  - 장기적으로 살아남을 가능성이 높다고 평가

### 느낀점

- 언어는 계속 생기지만 큰 혁신은 점점 줄어드는 느낌
- 다른 학문처럼 표준화되거나 어느 정도 통일되는 방향도 괜찮아 보임

## 17장 AI

- 인간의 뇌
  - 최신 마이크로칩과 비교해도 훨씬 복잡하고 처리 능력이 뛰어남
  - 현재 기술로는 인간의 뇌를 완전히 따라잡기 어려움

- 신경망
  - 생물학적 뉴런 구조를 모방한 AI의 핵심 아이디어
  - 층과 가중치를 통해 학습하며, 데이터를 기반으로 결과 도출
  - 결정론적이지 않고 확률적 접근(운에 가까운 요소 포함)
  - 전통적인 프로그래밍이 아닌 엔지니어링에 가까움
  - 프로그래머는 명확한 결과를 다루지만, AI는 애매한 문제 해결을 돕는 도구

- 대규모 언어 모델(LLM)
  - 신경망 기반 + 확률적 모델
  - 결과는 인상적이지만 지능이나 창의성을 가진 것은 아님
  - 완벽하진 않지만 충분히 유용한 도구

- 대규모 모델과 프로그래머
  - 프로그래머를 대체하기보다는 역할을 변화시킴
  - LLM도 C, 클로저, IDE처럼 하나의 도구일 뿐
  - 과거 컴파일러 등장 때와 마찬가지로, 오히려 더 많은 개발자가 필요해질 가능성
  - 결국 프로그래머는 디테일을 다루는 존재로 계속 필요함

### 느낀점

- AI가 강력해도 결국 확률 기반이라 완전한 대체는 어렵다는 점이 흥미로웠음
- 새로운 기술이 나올 때마다 역할이 사라지는 게 아니라 바뀐다는 흐름
- 결국 개발자는 계속 필요하고, 도구를 어떻게 쓰느냐가 더 중요해보임

## 18장 하드웨어

- 하드웨어는 초창기부터 빠르게 발전하며 성능이 크게 향상됨

- 무어의 법칙
  - 반도체 칩에 들어가는 부품 수가 매년 두 배씩 증가한다는 예측  
    = 초기에는 실제로 성능 향상을 이끌었지만 점차 물리적 한계에 도달함  
    = 최근에는 성장 속도가 둔화되며 정체되는 흐름

- 양자컴퓨터
  - 기존 반도체의 한계를 넘을 새로운 기술로 제시됨

### 느낀점

- 끝없이 발전할 것 같던 하드웨어가 현재에 와서는 한계에 도달..?
